# Экосистема прогрева, удержания и продаж media.dmikhin.ru

Документ фиксирует систему по ТЗ из файла "Проектирование экосистемы.pdf". Принцип: эволюция существующего журнала, а не новый сайт.

## 1. Карта экосистемы

Базовая логика:

1. Человек приходит на статью, главную, хаб, диагностику, Stepik или из соцсетей.
2. Сайт не оставляет его в тупике: предлагает следующий материал, хаб, лид-магнит, Telegram, курс или консультацию.
3. Продажа не давит: курс появляется как продолжение темы, консалтинг - как разбор конкретной ситуации.
4. Возврат строится через Telegram, VK, сохранение материалов, подборки, серии и обновляемые хабы.

Основная воронка:

```text
Статья
-> Следующая статья
-> Подборка материалов
-> Тематический хаб
-> Лид-магнит или диагностика
-> Telegram / VK
-> Курс
-> Кейс
-> Консультация
```

## 2. Карта пользовательских сценариев

Основные входы:

- Яндекс: SEO-статья -> продолжить маршрут -> хаб -> лид-магнит -> Telegram.
- Google: статья -> похожие материалы -> курс по теме.
- Telegram: новая статья -> реакция -> сохранение -> хаб.
- VK: короткий анонс -> статья -> диагностика.
- Рекомендация: главная -> об авторе -> с чего начать -> практика изменений.
- Stepik: курс -> журнал -> хаб -> Telegram.
- Диагностика: форма -> результат -> PDF -> отправить Денису -> консультация.
- Запрос про ИИ: промты -> хаб ИИ -> курс -> консультация.
- Запрос про управление: статья -> диагностика -> хаб Управление -> курс.
- Запрос про карьеру: карьерная статья -> хаб Карьера -> маршрут -> курс.

## 3. Карта доверия

Доверие усиливают:

- Авторская позиция Дениса: управление, продажи, ИИ и системные изменения.
- Хабы как система знаний, а не лента случайных публикаций.
- Практика изменений: кейсы без раскрытия конфиденциальности.
- Диагностики с персональным результатом.
- Курсы как образовательные маршруты, связанные с задачами читателя.
- Мягкие CTA вместо агрессивной продажи.

## 4. SEO-архитектура

Приоритетные кластеры для Яндекса:

- Управление: управляемость бизнеса, регулярный менеджмент, эффективный руководитель.
- Проекты: управление проектами, риски, портфель проектов, delivery.
- ИИ: ИИ в управлении, промты для руководителя, внедрение ИИ.
- Системное мышление: корневые причины, системный анализ, ограничения.
- Agile и трансформации: управление изменениями, Agile-трансформация, закрепление изменений.
- Карьера: рост специалиста, переход в руководители, управленческие навыки.
- Продажи: точки роста продаж, система продаж, коммерческий эффект изменений.

Реализация в коде:

- `lib/ecosystem.ts` содержит SEO-карту для каждого хаба.
- `app/hub/[hub]/page.tsx` использует SEO-title, description, поисковые запросы и внутренние ссылки.

## 5. Лид-магниты

Минимальный набор:

- Диагностика управляемости бизнеса.
- 50 промтов для руководителя.
- Шаблон стратегической сессии.
- Карта рисков.
- Шаблон OKR.
- Матрица приоритизации.
- Чек-лист внедрения ИИ.
- Чек-лист управленческого хаоса.
- Инструменты системного мышления.

Логика показа:

- Управление, собственники, CEO -> диагностика управляемости.
- ИИ -> 50 промтов и чек-лист внедрения ИИ.
- Проекты -> карта рисков и матрица приоритизации.
- Трансформации -> шаблон стратегической сессии.
- Системное мышление -> инструменты системного разбора.

## 6. Система CTA

Мягкая лестница:

- После первого экрана статьи: сохранить материал или открыть содержание.
- После статьи: продолжить маршрут.
- После 2 материалов: открыть хаб.
- После хаба: скачать лид-магнит или пройти диагностику.
- После диагностики: отправить результат Денису или обсудить вывод.
- После лид-магнита: подписаться на Telegram.
- После страницы курсов: перейти на Stepik.
- После кейса: обсудить похожую ситуацию.

## 7. Прогрев к курсам

Принцип: не продавать курс как баннер, а показывать его как продолжение темы.

- Управление -> "Эффективный руководитель".
- Проекты -> "Управление проектами".
- ИИ -> "Промт-инжиниринг".
- Трансформации -> Agile / AI / изменения.
- Системное мышление -> продукт и система.
- Карьера -> управленческая роль и рост.

## 8. Прогрев к консалтингу

Маршрут:

```text
Проблема
-> Статья
-> Кейс
-> Диагностика
-> Разбор ситуации
-> Консультация
```

Правило: консалтинг не продавать напрямую. CTA должен звучать как "обсудить похожую ситуацию", "разобрать результат диагностики", "найти ограничение", "собрать маршрут изменений".

## 9. Система возврата

Механики:

- Telegram и VK как каналы повторного касания.
- Сохранение статьи.
- Обновляемые хабы.
- Серии материалов.
- Новые материалы в тематических маршрутах.
- PDF-результаты диагностик.
- Подборки после лид-магнитов.

## 10. Метрики

Отслеживать:

- Глубина просмотра: цель 2.2-3.0 страницы за визит.
- Время на сайте: цель 3-5 минут для статей.
- Переходы между статьями: цель 12-25% дочитавших.
- Переходы в хабы: цель 8-15% с главной и статей.
- Скачивания лид-магнитов: цель 3-8% с посадочных.
- Подписки и переходы в Telegram/VK: цель 5-12% вовлеченных читателей.
- Переходы на Stepik: цель 4-10% с карточек курсов.
- Заявки на консультацию: цель 1-4% с диагностики и кейсов.
- Поисковый трафик Яндекса: цель рост 10-20% месяц к месяцу после индексации хабов.
